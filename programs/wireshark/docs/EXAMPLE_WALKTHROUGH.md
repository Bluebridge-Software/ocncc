## Example 1: Wallet General Recharge Message

### Message Structure

This example shows a typical OCNCC Wallet General Recharge (WGR) message structure:

```
Message (Top-level Map)
├─ HEAD (Header Map)
│  ├─ VERS: 1 (Int)
│  ├─ TIME: 2024-02-26 12:30:00 (Date)
│  ├─ TYPE: 'WGR ' (Symbol)
│  └─ SEQN: 12345 (Int)
│
├─ BODY (Payload Map)  
│  ├─ ACCT: "447700900123" (String)
│  ├─ IMSI: "234150000000001" (String)
│  ├─ AMNT: 1000 (Int)
│  ├─ CURR: 'GBP ' (Symbol)
│  └─ RCID: "RCHRG-2024-001" (String)
│
└─ TAIL (Trailer Map)
   ├─ STAT: 'OK  ' (Symbol)
   └─ CODE: 0 (Int)
```

### Using Display Filters

Find all wallet recharge messages:
```
escher && escher.map.symbol contains "WGR"
```

Find messages with specific amounts:
```
escher && escher.map.symbol == "AMNT" && escher.int == 1000
```

Find successful transactions:
```
escher && escher.map.symbol == "STAT" && escher.symbol contains "OK"
```

## Example 2: Simple Test Message

### Source Code

```python
message = {
    'TEST': {
        'NULL': None,
        'INT ': 42,
        'FLOT': 3.14159,
        'STR ': 'Hello World',
        'DATE': {'__date__': datetime(2024, 1, 1, 0, 0, 0)},
        'SYMB': {'__symbol__': 'TEST'}
    }
}
```

### Hex Dump

File: `simple_test.escher`

```
00000000: 0044 0001 5445 4e00 003c 0006 4e55 4000  .D..TEN..<..NU@.
00000010: 494e 4200 464c 4801 5354 4a03 4441 4406  INB.FLH.STJ.DAD.
00000020: 5359 4607 0000 002a 4009 21f9 f01b 866e  SYF....*@.!....n
00000030: 0b48 656c 6c6f 2057 6f72 6c64 6592 0080  .Hello World....
00000040: 5445 5354                                TEST
```

### Decoding Steps

1. **Read outer map header:**
   - Length: 0x0044 (68 bytes)
   - Items: 0x0001 (1 entry)

2. **Read first key (offset 4):**
   - Key: 0x5445 4e00
   - Symbol: TEST (bits 31-13)
   - Type: MAP_TYPE (bits 12-9)
   - Offset: 0 (bits 8-0)

3. **Read inner map at offset 8:**
   - Length: 0x003c (60 bytes)
   - Items: 0x0006 (6 entries)

4. **Read inner map keys:**
   - NULL: type NULL_TYPE, offset 0 → no data
   - INT : type INT_TYPE, offset 0 → read int32 at data[0]
   - FLOT: type FLOAT_TYPE, offset 4 → read float64 at data[4]
   - STR : type STRING_TYPE, offset 12 → read string at data[12]
   - DATE: type DATE_TYPE, offset 24 → read timestamp at data[24]
   - SYMB: type SYMBOL_TYPE, offset 28 → read symbol at data[28]

## Example 3: Array Test Message

### Structure

```
Message
└─ ARRY: [1, 2, 3, "item", null, 'TEST']
```

### Hex Dump

File: `array_test.escher`

```
00000000: 0030 0001 4152 4c00 0028 0006 0200 0201  .0..ARL..(...... 
00000010: 0202 0a03 0005 0600 0000 0001 0000 0002  ................
00000020: 0000 0003 0469 7465 6d00 0000 5445 5354  .....item ...TEST
```

### Array Index Decoding

Array index entries are 16-bit values:

1. Index 0: 0x0200
   - Type: INT_TYPE (0x02 >> 1 = 0x01)
   - Offset: 0

2. Index 1: 0x0201
   - Type: INT_TYPE
   - Offset: 4 bytes

3. Index 2: 0x0202
   - Type: INT_TYPE
   - Offset: 8 bytes

4. Index 3: 0x0a03
   - Type: STRING_TYPE (0x0a >> 1 = 0x05)
   - Offset: 12 bytes

5. Index 4: 0x0005
   - Type: NULL_TYPE (0x00)
   - Offset: 20 bytes (unused)

6. Index 5: 0x0600
   - Type: SYMBOL_TYPE (0x06 >> 1 = 0x03)
   - Offset: 24 bytes

## Example 4: Extended Format Message

For messages larger than 2048 bytes, the extended format is used:

```
Header:
00 FE FF  - Extended header marker (0xFFFE)
01        - Control block length (1 byte)
00        - Control block data
00 00 10 00 - Total length (4096 bytes)
00 00 00 64 - Number of items (100)
[Index entries...]
[Data...]
```

### Detecting Extended Format

In Wireshark:
```
escher.extended == 1
```

In Lua dissector:
```lua
if header_id == 0xFFFE then
    -- Extended format
    byte_length = buffer(offset+4, 4):uint()   -- 32-bit
    num_items = buffer(offset+8, 4):uint()     -- 32-bit
end
```

## Troubleshooting Examples

### Problem: Message not decoding

**Symptoms:**
- Raw hex displayed
- No ESCHER protocol shown

**Solution:**
1. Right-click packet
2. "Decode As..."
3. Select "ESCHER"
4. Click OK

### Problem: Incorrect values shown

**Check these:**
```
1. Byte order - should be big-endian
2. Alignment - values at 4-byte boundaries
3. String length encoding:
   - If first byte < 128: 1-byte length
   - If first byte >= 128: 2-byte length with bit 15 set
```

### Problem: Dirty flag set

**What you see:**
```
Dirty Flag: true
Pointer Value: 0xbf80bd21
```

**Explanation:**
The message was unpacked in memory by the source application. This is a pointer to an in-memory C++ structure, not encoded data.

**Solution:**
Capture the message before it's processed by the application, or ensure the source application calls `pack()` before transmission.

## Creating PCAP Files for Testing

### Method 1: Using text2pcap

Create a hex text file (`message.txt`):
```
0000  00 44 00 01 54 45 4e 00  00 3c 00 06 4e 55 40 00
0010  49 4e 42 00 46 4c 48 01  53 54 4a 03 44 41 44 06
...
```

Convert to PCAP:
```bash
text2pcap -l 147 message.txt message.pcap
```

### Method 2: Using Python and Scapy

```python
from scapy.all import *

# Create Escher message
escher_data = open('wallet_recharge.escher', 'rb').read()

# Create packet
packet = Ether()/IP(dst="10.0.0.2")/TCP(dport=5000)/Raw(load=escher_data)

# Save to PCAP
wrpcap('test.pcap', packet)
```

### Method 3: Using od and text2pcap

```bash
# Create hex dump
od -A x -t x1 -v wallet_recharge.escher > message.hex

# Convert to PCAP  
text2pcap -l 147 -t "%Y-%m-%d %H:%M:%S." message.hex message.pcap
```

## Common Patterns

### Pattern 1: Request-Response

Request:
```
HEAD.TYPE = 'REQ '
HEAD.SEQN = 123
BODY = {...}
```

Response:
```
HEAD.TYPE = 'RSP '
HEAD.SEQN = 123  # Match request
TAIL.STAT = 'OK  '
TAIL.CODE = 0
```

Filter for matching pairs:
```
escher.int == 123 && escher.map.symbol contains "SEQN"
```

### Pattern 2: Event Notification

```
HEAD.TYPE = 'EVNT'
BODY.EVTY = 'RCHR'  # Recharge event
BODY.TIME = <timestamp>
BODY.DATA = {...}
```

### Pattern 3: Batch Operations

```
HEAD.TYPE = 'BTCH'
BODY.OPER = 'UPDT'
BODY.ITMS = [item1, item2, ...]  # Array of operations
```

## Integration with Other Tools

### With tshark (CLI)

```bash
# Extract all Escher messages
tshark -r capture.pcap -Y escher -T json

# Count Escher packets
tshark -r capture.pcap -Y escher -T fields -e frame.number | wc -l

# Extract specific field
tshark -r capture.pcap -Y escher -T fields -e escher.int
```

### With Python/pyshark

```python
import pyshark

cap = pyshark.FileCapture('capture.pcap', 
                          display_filter='escher')

for packet in cap:
    if hasattr(packet, 'escher'):
        print(f"Escher packet: {packet.escher}")
```
